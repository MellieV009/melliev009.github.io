Taken P2-K2

De kerntaak heeft twee werkprocessen waarin wordt verwacht dat de kandidaat in staat is om kwetsbaarheden te identificeren binnen een organisatie en/of informatievoorziening, én de kandidaat moet aanvallen/incidenten kunnen afwenden.

Geeft security advies en verbetert de security WP1

Je moet iets vinden wat fout is en je moet advies geven. Je gaat dus tools gebruiken om dingen op te sporen, en een analyse maken. Waarschijnlijk moet je hier specifieke Sysinternals programma's gaan gebruiken, zoals Procexp, Procmon, Autoruns en TCPView. Volgens open source data draait hier een platform van drie VM's, een domain-controller, een SQL-server en een werkstation. In het eerste gedeelte van het examen dient de omgeving uitgebreid geanalyseerd te worden. Verder krijg je ook dingen aangeboden. Zo zijn er een paar korte interviews gehouden door een security-collega met enkele werknemers van het bedrijf. De interviews zijn genotuleerd en de kandidaat wordt in de gelegenheid gesteld om die interviews te bestuderen. Uit deze interviews vallen een aantal zaken op die er op duiden dat er kwetsbaarheden zijn binnen de IT-omgeving en/of het gebruik hiervan. Op de VM wordt een toolbox ter beschikking gesteld met verschillende security-tools waarmee de IT-omgeving op een aantal punten getest kunnen worden.

Reageert op security incidenten WP2

Volgens open source data dient een gerapporteerd incident geanalyseerd te worden én er wordt een lijst met gedetecteerde zwakheden op de SQL-server aangereikt met het verzoek om de server op die punten te beveiligen. Het vermoeden wordt uitgesproken dat er vertrouwelijke gegevens uit een database zijn gelekt, jij moet dat gaan onderzoeken. Met behulp van een sjabloon kan de kandidaat de bevindingen onderverdelen per asset. De kandidaat selecteert uit een organogram de juiste personen die hiervan op de hoogte moeten worden gesteld en stelt daarvoor e-mails op. De kandidaat krijgt een document aangereikt met een aantal incidenten (gevonden zwakheden) op de SQL-server. Het gaat dan bijvoorbeeld om verdachte software, onverwachte externe toegang en zwakheden in het accountbeheer. Deze zwakheden moeten beveiligd worden. Hiervoor dienen aanpassingen gedaan te worden op de VM. Alle verrichte aanpassingen dienen gedocumenteerd te worden.

Foute dingen

  • Er is data uit de database gehaald
    • FIX: Controleer dit in de General Query Log (zoek iets met LIMIT 1000)
  • Windows Securtiy firewall is uit
    • FIX: Aanzetten
  • Windows Defender is uit
    • FIX: Aanzetten
  • In de downloads staat NicePics.txt en dit is een Eicar file
  • In de downloads staat PortQry.exe en dit is een raar programma om iets met RPC te doen fzo idk
  • PC is geïnfecteerd
    • FIX: Het virus is NetCat Remote Tool (nc.exe). Dit staat in een hidden folder in C:\LP. Dit mapje verwijderen dan is het virus weg, er is een exclusion in Defender voor dit virus in Regedit! Gebruik hier vanuit Sysinternals de programma's Process Explorer om het virus te zien, en Autoruns om het virus uit te zetten bij opstarten. Gebruik VirusTotal om het virus te analyseren
  • Virus op de server verwijderen
    • FIX: Zet antivirus aan en scan, eventueel gebruik Malwarebytes/HitmanPro
  • UltraVNC staat op de server geïnstalleerd
    • FIX: Uninstallen
  • MySQL Workbench root account wachtwoord is root
    • FIX: Wachtwoord aanpassen
  • Wachtwoorden van gebruikers zijn te makkelijk en gebruiken geen encryptie
    • Fix: Gebruik Strong Password Encryption om makkelijke wachtwoorden te voorkomen en caching_sha2_password om veilig wachtwoorden te encrypten

Port en CVEDetails uitleg

Let op: Niet iedere poort is gelijk een virus/slecht beveiligd, en niet iedere service die op een poort loopt is gelijk kwetsbaar. Het is pas kwetsbaar als er een exploit beschikbaar is die onder de juiste versie valt, zodat het product dus affected is (gebruik CVEDetails). Poorten met als service HTTP kun je bezoeken in je browser. Sommige poorten staan nou eenmaal open, zoals SSH, FTP, HTTP etc. In kleine woorden; trek niet snel een oordeel en verzamel info!
